# FPGA-ArtyA7 — Arty A7-100T Implementation

This directory contains the Vivado project and build infrastructure for targeting the **Digilent Arty A7-100T** FPGA board.

## Project

| Field        | Value               |
|--------------|---------------------|
| Project Name | `Drop-In-JTAG`      |
| Target Board | Arty A7-100T        |
| FPGA Device  | Xilinx XC7A100T     |
| Toolchain    | Xilinx Vivado       |

## Directory Structure

```
FPGA-ArtyA7/
├── Makefile                     # Build automation
├── build_arty_project.tcl       # Full implementation TCL script
├── synth_arty_project.tcl       # Synthesis-only TCL script
└── Drop-In-JTAG/                # Generated Vivado project (after build)
    └── Drop-In-JTAG.runs/
        ├── synth_1/             # Synthesis outputs
        └── impl_1/              # Implementation & bitstream outputs
```

> The `Drop-In-JTAG/` project directory is generated by Vivado and is not checked into source control.

## Prerequisites

- Xilinx Vivado (with Arty A7-100T board files installed)
- `vivado` available on your `PATH`, or override with `VIVADO=/path/to/vivado`

## Build

```bash
# Full build: synthesis + implementation + bitstream
make

# Synthesis only
make synth

# Open the project in the Vivado GUI
make open
```

## Cleaning Up

```bash
# Remove logs and temporary files
make clean

# Remove logs and the generated Vivado project
make distclean

# Aggressive cleanup (bitstreams, reports, caches, etc.)
make nuke
```

## Output

After a successful `make`, the bitstream and related outputs are located in:

```
Drop-In-JTAG/Drop-In-JTAG.runs/impl_1/
```

## Swapping the Top-Level for FPGA (MMCM Clock)

The simulation and FPGA top-level modules differ: the FPGA version instantiates
an MMCM-based clock generator (`clk_gen.sv`) required for the Arty A7-100T,
while the simulation version uses a plain clock input. The `swap_top.sh` script
manages switching between the two.

```
FPGA-ArtyA7/
├── swap_top.sh      # Swap utility
├── top_fpga.sv      # FPGA top-level (instantiates MMCM via clk_gen)
├── top_orig.sv      # Original simulation top-level
└── clk_gen.sv       # MMCM clock generation module (Arty A7-100T)
```

The script copies the chosen `top.sv` into `../JTAG-HDL/`. When installing the
FPGA version, `clk_gen.sv` is copied alongside it. When restoring the original,
`clk_gen.sv` is removed from the target directory.

### Usage

```bash
# Interactive menu
./swap_top.sh

# Non-interactive
./swap_top.sh --fpga      # Install FPGA version (top_fpga.sv + clk_gen.sv)
./swap_top.sh --orig      # Restore simulation version (top_orig.sv)
./swap_top.sh --status    # Show which version is currently active
```

> **Before running `make`**, ensure the FPGA top-level is active:
> ```bash
> ./swap_top.sh --fpga
> ```
> You can confirm the active version at any time with `./swap_top.sh --status`.

## Sipeed USB-JTAG Connections (Pmod JA)

The JTAG signals are routed to the **Pmod JA** connector on the Arty A7-100T.
Connect the Sipeed USB-JTAG/TTL debugger as follows:

```
Pmod JA (top row)                    Pmod JA (bottom row)
  Pin1   Pin2   Pin3   Pin4   Pin5   Pin7   Pin8   Pin9   Pin10  Pin11
  JA1    JA2    JA3    JA4    GND    JA7    JA8    JA9    JA10   GND
  TDI    TDO    TRST   TMS                                 TCK
```

| Sipeed Pin | Signal | Pmod JA Pin | Row         |
|------------|--------|-------------|-------------|
| TDI        | TDI    | JA1         | Top, 1st    |
| TDO        | TDO    | JA2         | Top, 2nd    |
| TMS        | TMS    | JA4         | Top, 4th    |
| TCK        | TCK    | JA10        | Bottom, 4th |
| GND        | GND    | JA5 or JA11 | Either, 5th |
| **3.3V**   | VCC    | JA6 or JA12 | Either, 6th |

> **Note on TRST:** The Sipeed USB-JTAG does not expose a dedicated TRST pin.
> Connect JA3 (TRST) to 3.3V through a 10KΩ pull-up resistor to keep the TAP
> out of reset, or leave it unconnected if your software drives TRST via the
> TMS reset sequence.

---

> ⚠️ **VOLTAGE WARNING — READ BEFORE CONNECTING**
>
> The Sipeed USB-JTAG/TTL exposes both a **3.3V** and a **5V** power pin.
> The Arty A7-100T Pmod I/O is **3.3V LVCMOS only**.
>
> **You MUST use the 3.3V pin. Connecting 5V will damage or destroy the FPGA I/O.**
>
> It is solely the user's responsibility to verify correct voltage selection
> before powering on. No warranty is expressed or implied for damage resulting
> from incorrect connections.
